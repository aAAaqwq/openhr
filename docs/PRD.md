## 一句话

基于 Boss 直聘的 AI 自动化招聘系统 —— 从打招呼到约面试全流程自动化，简历自动解析入飞书多维表格。

## 背景

招聘高频重复工作（筛选候选人 → 打招呼 → 获取简历 → 沟通面试 → 确认时间地点）完全可由 AI 驱动自动化，节省 HR 大量时间。

### M1: 登录模块
- 检测登录状态（cookie/页面元素）
- 未登录 → 生成二维码 → 通知管理员扫码
- 登录态持久化（`data/cookies/boss_cookies.json`）
- 登录失效自动检测 + 重新触发二维码

### M2: 打招呼模块
- 自动浏览推荐候选人列表（`zhipin.com/web/boss/recommend`）
- 按岗位需求筛选候选人（`config/filters.json`）
- 发送个性化打招呼消息（`config/templates.json`）
- 每日打招呼次数追踪 + 上限控制（默认 100 次/天）
- 操作间隔 5-15s 随机化（调用 M6）

### M3: 简历解析 + 飞书多维表格
- 获取候选人简历信息（聊天窗口/简历卡片/PDF）
- 解析字段：姓名、年龄、学历、工作经历、技能、期望薪资
- LLM 评分（1-10 综合匹配度）+ 200 字评价报告
- 飞书多维表格写入（去重：手机号/姓名+公司）
- 字段映射：简历字段 → 多维表格列（`config/feishu.json`）

### M4: 智能聊天跟进
- 主动遍历 Boss 聊天栏所有会话（`zhipin.com/web/boss/chat`）
- 结合岗位需求 + 候选人背景，LLM 生成回复
- 状态机：打招呼 → 兴趣确认 → 面试意向 → 时间地点确认
- 异常处理：候选人拒绝 → 礼貌结束 → 记录原因
- 每 15 分钟遍历一次聊天栏

### M5: 知识库系统
- 话术库：`data/knowledge/greetings.json` + `chat_patterns.json`
- 岗位需求库：`data/knowledge/position_requirements.json`
- 反馈模板：`data/knowledge/feedback_templates.json`
- 支持从历史聊天记录提取话术模式

### M6: 风控对抗
- 行为模拟：随机操作间隔、鼠标轨迹模拟、滚动行为
- 频率控制：打招呼间隔 5-15s 随机、每日上限硬限
- 验证码处理：检测到验证码 → 暂停 → 通知人工处理
- 封号预警：异常检测 → 自动降速/暂停 → 通知
- Playwright stealth 配置 + 浏览器指纹管理

## 技术栈

| 技术 | 用途 | 版本/说明 |
|------|------|-----------|
| **Playwright** | 浏览器自动化（必须 headed 模式） | Python async API |
| **playwright-stealth** | 反检测 | JS 注入反指纹 |
| **GLM-5-plus** | LLM 聊天生成 + 简历解析 | via zai provider（OpenAI 兼容格式） |
| **飞书 Bot API** | 多维表格写入 | `tenant_access_token` 认证 |
| **JSON** | 本地数据存储 | cookies / chat_logs / knowledge |

### 核心依赖
```bash
playwright + playwright-stealth  # 浏览器自动化
openai (Python SDK)              # LLM 调用（兼容 zai provider）
requests                         # 飞书 API 调用
```

## 人工介入断点

| 断点 | 模块 | 类型 | 触发条件 | 处理方式 |
|------|------|------|---------|----------|
| 首次登录扫码 | M1 | 🔴 必须 | Cookie 不存在或过期 | 截图 → 通知 → 等待扫码 |
| 筛选条件配置 | M2 | 🟡 可选 | 首次运行 | 对话式引导，有默认值 |
| 面试确认 | M4 | 🟡 可选 | 候选人同意面试 | 通知管理员确认 |
| 低分候选人 | M3 | 🟡 可选 | 评分 < 5 | 标记「待确认」 |
| 验证码失败 | M6 | 🔴 必须 | 自动解决失败 | 暂停 + 通知 |
| 封号预警 | M6 | 🔴 必须 | 异常行为检测 | 立即暂停 + 通知 |

**自动化率**：~85% 全自动，~10% 可选确认，~5% 必须人工。

## 风险评估

| 风险 | 概率 | 影响 | 对策 |
|------|------|------|------|
| Boss直聘封号 | 中 | 高 | 风控模块 + 频率限制 + 行为模拟 |
| 页面结构变更 | 低 | 中 | CSS 选择器可配置化（`config/selectors.json`） |
| 验证码阻断 | 中 | 中 | 自动检测 + 暂停 + 人工介入 |
| LLM 回复质量 | 低 | 中 | 知识库话术兜底 |
| 飞书API限流 | 低 | 低 | 批量写入 + 重试 + 指数退避 |
